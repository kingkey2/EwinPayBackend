加密方式

加密方式皆为SHA256

加密后需再转为大写

php:

        
                hash('sha256', $data, false);

java:

        
                MessageDigest digest = MessageDigest.getInstance("SHA-256");
                digest.reset();
                digest.update(input.getBytes("utf8"));
                toReturn = String.format("%064x", new BigInteger(1, digest.digest()));

加密字串:

5566520

加密后结果:

69481E4583F7A8FBC1200BC629603CB9A0D5B16822DE87B0ACA965C89CD80F00

线上测试网址:https://passwordsgenerator.net/sha256-hash-generator/

支付申请

URL: RequirePayment

Method: Post

Content-Type: application/x-www-form-urlencoded

Request :

PS : 签章使用SHA256加密。

请求的类型是 form，即 Content-Type: application/x-www-form-urlencoded
PostForm submit 导页到支付页面即可。

Parameter Type Required Description
CompanyCode String 是 商户代码
CurrencyType String 是    币别(详见对照表)
ServiceType String 是    服务类型(详见对照表)
BankCode String 否    银行代码(详见对照表)
ClientIP Integer 是 提单会员IP
OrderID String 是 订单编号自行产生后带入
OrderDate String 是 订单日期
字串格式要求为 yyyy-MM-dd HH:mm:ss ex:2018-01-01 11:22:33

只支援整数
ReturnURL String 是 回传网址
UserName String 否 汇款人户名
如果为实名制渠道需填写
State String 否 公用回传参数

代付订单确认

URL: 商户需提供给我方客服设定

PS :

1 : 此为代付申请确认 API,我方收到 代付申请 后,会呼叫商户提供的网域,确认是否有该笔订单申请。

2 : 请提供查询订单网域由我方设定。

3 : 订单查询失败时,回传资讯请填写清楚错误资讯。

4 : API接口格式如下 :

Method: POST

Content-Type: application/json

5 : 请求参数与回传格式如下 :

请求参数(我方呼叫时所带的参数) :

参数名称 参数变量名 类型 是否必填 说明
商户代码 CompanyCode String Y 商户代码
商户订单号 OrderID String Y 商户下单订单号

回传参数(商户收到请求后返回是否有此笔订单) :

参数名称 参数变量名 类型 是否必填 说明
成功與否的狀態 status String Y true:成功(确认有此订单) false:失败
讯息 msg String Y 讯息(成功或失败都请提供资此参数)

示例 :

1 : 确认有此笔订单。

我方请求参数 : 

        
                {
                    "CompanyCode": "AAA",
                    "OrderID": "WT12345"
                }

商户回传参数 : 

        
                {
                    "status": "true",
                    "msg": "有查询到此笔订单资讯,订单号: WT12345"
                }

2 : 查询失败时 :

我方请求参数 : 

        
                {
                    "CompanyCode": "AAA",
                    "OrderID": "WT12345"
                }

商户回传参数 : 

        
                {
                    "status": "false",
                    "msg": "请提供相关错误资讯"
                }

快速代付

快速代付，为代付功能的快速版本，提供代付输入的画面

  1. 带入参数开启快速支付页面
  2. 会员填写完资讯后，会呼叫 人工审核接口 人工审核接口须由贵司提供。
  3. 审核完毕后，将结果传入 支付训习通知 我方确认完毕后,即完成代付申请。
    PS : 如贵司传送结果为审核失败，则该单流完完毕，不需等待 代付异步回传结果，
    如贵司传送结果为审核成功，我方确认参数无误后，处理完毕后会回传结果通知 代付异步回传结果。
